  • -get:

    Method to GET a url with the a path.

    This method uses the base url and adds a path comment to that url so we never mutate the base URL.

    Called like so:

    @code // i.e network = http://api_url.io/API TRLRequest *req = [network get:@products]; // req URL = http://api_url.io/API/products @endcode

    Note

    By default this passes an empty Parameters dictionary, as the end user may wish to add things later on that they didn’t know they could do before.

    Declaration

    Objective-C

    - (nonnull TRLRequest *)get:(nonnull NSString *)path;

    Parameters

    path

    The path you wish to append

    Return Value

    A TRLRequest instance.

  • -get:parameters:

    Method to GET a url with the a path and Parameters.

    This method uses the base url and adds a path comment to that url so we never mutate the base URL.

    Called like so:

    @code // i.e network = http://api_url.io/API TRLRequest *req = [network get:@products parameters:{@foo:@bar}]; // req URL = http://api_url.io/API/products?foo=bar @endcode

    Note

    By default this passes the parameters use @c[TRLURLEncoding defaultEncoding]

    Declaration

    Objective-C

    - (nonnull TRLRequest *)get:(nonnull NSString *)path
                 parameters:(Parameters *_Nullable)parameters;

    Parameters

    path

    The path you wish to append
    parameters

    The Parmeters for the URL Request, can be set to NULL. if thats the case please use @c-[get:]

    Return Value

    A TRLRequest instance.

  • -get:parameters:encoding:

    Method to GET a url with the a path, parameters and the encoding for said parmeters.

    This method uses the base url and adds a path comment to that url so we never mutate the base URL.

    Called like so:

    @code // i.e network = http://api_url.io/API TRLRequest *req = [network get:@products parameters:{@foo:@bar} encoding:[TRLURLEncoding defaultEncoding]]; // req URL = http://api_url.io/API/products?foo=bar @endcode

    Note

    By default this passes the HTTPHeaders as NULL. Accept-Encoding, Accept-Language and User-Agent are included in the request

    The Parmeters for the URL Request, can be set to NULL. if thats the case please use @c-[get:]
    encoding

    A TRLRequest instance.

  • -get:parameters:encoding:headers:

    Method to GET a url with the a path, parameters, the encoding for said parmeters and the HTTPHeaders.

    This method uses the base url and adds a path comment to that url so we never mutate the base URL.

    Called like so:

    @code // i.e network = http://api_url.io/API TRLRequest *req = [network get:@products parameters:@{@foo:@bar} encoding:[TRLURLEncoding defaultEncoding] headers:@{@From: @user@example.com}]; // req URL = http://api_url.io/API/products?foo=bar @endcode

    Note

    By default Accept-Encoding, Accept-Language and User-Agent are included in the request
